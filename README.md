## 🪟 How to Open the Labs

1. Make sure Cisco Packet Tracer is installed on your Windows PC.
2. Open the folder you extracted from the ZIP file.
3. Look for Packet Tracer lab files, often with a `.pkt` file name.
4. Double-click the lab file.
5. If Packet Tracer asks what app to use, choose Cisco Packet Tracer.
6. Wait for the lab to load.
7. Start working through the devices in the file.

If the file does not open, right-click it and choose **Open with**. Then select Cisco Packet Tracer.

## 🔧 Basic Use

After the lab opens, you can use Packet Tracer to click on devices and inspect their settings.

Common actions include:

- Click a PC to set its IP address
- Click a router to view ports and settings
- Use the connections tool to check links
- Open the command prompt on a PC
- Run a ping to test network access

These labs are meant to help you build a clear picture of how small networks work.

## 🧭 Suggested Practice Flow

If you are new to Packet Tracer, use this simple order:

1. Open one lab file.
2. Identify the devices on the screen.
3. Check which devices need setup.
4. Add IP settings where needed.
5. Confirm cable links are in place.
6. Test the network with ping.
7. Repeat the same steps in the next lab.

This process helps you learn the basics in a steady way.

## 🛠️ Troubleshooting

If something does not work, try these steps:

- Check that Packet Tracer is installed
- Confirm the lab file is fully extracted from the ZIP
- Make sure you are opening the correct `.pkt` file
- Retry the download if the file seems broken
- Use a newer version of Windows if your system is very old
- Close and reopen Packet Tracer if the file freezes

If a ping fails, check the IP address, subnet mask, and cable links first. These are the most common causes in basic labs.
